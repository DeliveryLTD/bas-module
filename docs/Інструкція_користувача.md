## 1. Вступ

**Delivery Connect for BAS** — це зовнішня обробка (`.epf`), яка підключає вашу облікову систему (БАС, BAF, 1С:Підприємство 8.3) до API служби доставки [Delivery Auto](https://delivery-auto.com.ua). Ви оформлюєте та відстежуєте доставки прямо з облікової програми, не переходячи в особистий кабінет на сайті.

**Кому підходить:** компаніям, що відправляють вантажі через Delivery Auto і ведуть облік у БАС/1С.

**Підтримувані конфігурації:**
- БАС Управління торговим підприємством 1.2 (звичайні форми)
- БАС Бухгалтерія 2.1 (керовані форми)
- Платформи: BAF 8.3.19+, 1С:Підприємство 8.3.x

> Обробка універсальна — автоматично визначає тип форм залежно від режиму застосунку. Окреме встановлення для різних конфігурацій не потрібне.

## 2. Встановлення

1. Завантажте файл `DeliveryConnect-BAS.epf` зі [сторінки релізів](https://github.com/DeliveryLTD/bas-module/releases/latest).
2. Відкрийте вашу облікову систему (БАС/1С).
3. У меню: **Файл → Відкрити** (або **Файл → Відкрити...**).
4. Оберіть завантажений файл `DeliveryConnect-BAS.epf`.

`[📷 Скріншот: меню Файл → Відкрити]`

5. Обробка відкриється. Якщо це перший запуск — перейдіть до **Налаштувань** (розділ 3).

> **Порада:** збережіть обробку у зручному місці на диску. Щоб не відкривати щоразу через меню, можна додати її до зовнішніх обробок вашої конфігурації (якщо адміністратор це дозволяє).

## 3. Налаштування

Перед першим використанням потрібно ввести ключі доступу до API Delivery Auto.

1. Натисніть кнопку **Налаштування** (жовта, угорі форми).

`[📷 Скріншот: форма Налаштувань]`

2. Заповніть поля:

| Поле | Опис |
|---|---|
| **Базовий URL** | Адреса сервера API. За замовчуванням `www.delivery-auto.com` — змінювати не потрібно. |
| **Культура** | Мова відповідей API. За замовчуванням `uk-UA`. |
| **Логін** | Ваш логін особистого кабінету Delivery Auto. |
| **Пароль** | Пароль особистого кабінету. |
| **APIКлюч** | Публічний API-ключ (з особистого кабінету). |
| **Секретний ключ** | Секретний ключ для підпису запитів (HMAC). |

3. Натисніть **Перевірити з'єднання** — система має показати ваше ім'я та ID клієнта. Це підтверджує, що ключі правильні.

`[📷 Скріншот: успішна перевірка з'єднання]`

4. Натисніть **Зберегти**.

> **Де взяти ключі?** API-ключ і секретний ключ видаються в особистому кабінеті Delivery Auto. Якщо їх немає — зверніться до менеджера Delivery Auto.

---

## 4. Пошук квитанції

Швидкий перегляд статусу однієї квитанції за номером.

1. Перейдіть на вкладку **Пошук**.
2. У поле **Номер квитанції** введіть номер.
3. Натисніть **Знайти квитанцію**.
4. У полі **Результат** з'явиться інформація: статус, склади, вартість тощо.

`[📷 Скріншот: вкладка Пошук із результатом]`

---

## 5. Реєстр квитанцій

Перегляд усіх ваших квитанцій списком.

1. Перейдіть на вкладку **Реєстр**.
2. Натисніть **Завантажити з API** — таблиця заповниться вашими квитанціями.

`[📷 Скріншот: реєстр квитанцій]`

3. Колонки: номер, дати, склади, відправник/отримувач, вартість, статус, оплата, заборона, накладений платіж (НП).
4. **Подвійний клік** на рядку — відкриє вікно з повними деталями квитанції.

`[📷 Скріншот: деталі квитанції по подвійному кліку]`

---

## 6. Створення квитанції

Оформлення нової квитанції на доставку.

1. Натисніть **Створити квитанцію** (жовта кнопка вгорі).
2. Оберіть **Схему доставки**:

| Схема | Опис | Що заповнювати |
|---|---|---|
| **Склад-Склад** | Зі складу на склад | Склад відправлення + Склад отримання |
| **Двері-Двері** | Від дверей до дверей | Адреса забору + Адреса доставки |
| **Склад-Двері** | Зі складу до дверей | Склад відправлення + Адреса доставки |
| **Двері-Склад** | Від дверей на склад | Адреса забору + Склад отримання |

> Залежно від обраної схеми форма автоматично показує потрібні поля (склади або адреси).

`[📷 Скріншот: форма створення квитанції, схема Склад-Склад]`

### Заповнення полів

**Маршрут:**
- **Місто відправлення** — натисніть кнопку вибору (`...`), введіть частину назви міста, оберіть зі списку.
- **Склад відправлення** — після вибору міста оберіть склад зі списку.
- **Місто/Склад отримання** — аналогічно.
- **Відправник** — оберіть зі списку ваших відправників.

> **Адреси (для схем Двері-*):** вводьте у форматі **«Вулиця, Будинок, Квартира»** через коми. Наприклад: `Велика Васильківська, 15, 2`. Без ком API не розпізнає адресу.

**Отримувач:**
- **Отримувач (ПІБ)** — прізвище, ім'я, по батькові.
- **Телефон отримувача** — контактний телефон.
- **Контактна особа** — якщо відрізняється від отримувача.
- **Юридична особа** + **ЄДРПОУ** — для юросіб.

**Вантаж** (таблиця):
- Натисніть **Додати** — з'явиться рядок.
- Заповніть: **Кількість місць**, **Вага (кг)**, **Об'єм (м³)**.
- **Категорія вантажу** — оберіть для КОЖНОГО рядка (натисніть `...` у колонці).

> Категорія вантажу вказується окремо для кожного місця, бо в одній квитанції різні місця можуть містити різний вантаж.

3. Натисніть **Створити квитанцію**. У полі **Результат** з'явиться номер створеної квитанції та вартість.

`[📷 Скріншот: успішно створена квитанція]`

---

## 7. Розрахунок вартості

Дізнатися вартість доставки до оформлення.

1. На формі створення квитанції заповніть маршрут і вантаж (як у розділі 6).
2. Натисніть **Розрахувати вартість**.
3. У полі **Результат** з'явиться розбивка: транспорт, оформлення, додаткові послуги, разом, орієнтовна дата прибуття.

> Розрахунок **не створює** квитанцію — лише показує вартість. Працює для всіх схем доставки.

---

## 8. Друк документів

Друк квитанції та наклейок (стікерів) у PDF.

1. Перейдіть на вкладку **Реєстр**, завантажте квитанції.
2. Виділіть потрібну квитанцію.
3. Натисніть:
   - **Друк квитанції** — лист квитанції (PDF).
   - **Друк стікерів** — наклейки 95×95 мм (PDF).
4. Система запропонує зберегти PDF-файл. Збережіть і відкрийте для друку.

`[📷 Скріншот: кнопки друку в реєстрі]`

---

## 9. Заборона видачі та зняття через SMS

### Встановити заборону

1. У реєстрі виділіть квитанцію.
2. Натисніть **Заборонити видачу**.
3. Підтвердіть. Заборону встановлено — отримувач не зможе забрати вантаж без вашого дозволу.

### Зняти заборону (через SMS)

1. У реєстрі виділіть квитанцію із забороною.
2. Натисніть **Зняти заборону**.
3. На ваш телефон (вказаний в акаунті Delivery Auto) надійде **SMS-код**.
4. У вікні, що відкриється, введіть код із SMS.
5. Натисніть **Підтвердити** — заборону знято.

`[📷 Скріншот: вікно введення SMS-коду]`

> Якщо введено нечислові символи — система попередить: «Код з SMS має складатися лише з цифр».

## 10. Виклик кур'єра

Замовлення кур'єра для забору вантажу з вашої адреси.

1. Натисніть **Виклик кур'єра** (вгорі форми).
2. Заповніть:
   - **Контактна особа**, **Телефон** (обов'язково).
   - **Email** (необов'язково — якщо порожній, береться з вашого профілю).
   - **Область, Місто, Адреса** (обов'язково Місто й Адреса).
   - **Поверх**, **Місто доставки** (за потреби).
   - Параметри вантажу: **Дата забору**, **Вага**, **Об'єм**, **Кількість місць**, **Примітка**.
3. Натисніть **Відправити заявку**. Кур'єр зв'яжеться з вами.

`[📷 Скріншот: форма виклику кур'єра]`

## 11. Інтеграція з документами 1С

Оформлення квитанції на підставі документа вашої облікової системи (реалізація, рахунок).

1. Перейдіть на вкладку **Документи 1С**.
2. Оберіть **тип документа** (Реалізація товарів та послуг / Рахунок на оплату).
3. Вкажіть **період** (З / По).
4. Натисніть **Оновити** — завантажиться список документів за період.

`[📷 Скріншот: вкладка Документи 1С]`

5. Виділіть документ.
6. Натисніть **Оформити на підставі** — відкриється форма створення квитанції з уже підставленими даними контрагента (отримувач, телефон, номер документа).
7. Доповніть маршрут і вантаж, створіть квитанцію.

> Квитанція зв'язується з документом через номер — зручно для звірки.

## 12. Накладений платіж

Якщо отримувач має сплатити за товар при отриманні.

1. На формі створення квитанції, у блоці **Накладений платіж**:
   - Поставте галочку **Накладений платіж**.
   - Вкажіть **Суму**.
   - Оберіть **Розрахунковий рахунок** (натисніть `...`, оберіть зі списку ваших рахунків).
2. Створіть квитанцію.

`[📷 Скріншот: блок Накладений платіж]`

> **Важливо:** працює лише тип «розрахунковий рахунок». Якщо увімкнено НП, але не обрано рахунок — система не дасть створити квитанцію й попередить.

## 13. Поширені питання

**HTTP 401 / «Authorization has been denied»**
Неправильні ключі або логін/пароль. Перевірте Налаштування → Перевірити з'єднання.

**«Вкажіть адресу доставки» при заповненій адресі**
Адресу потрібно вводити через коми: «Вулиця, Будинок, Квартира».

**«Unrecognized Guid format»**
Не заповнено обов'язкове поле (місто, склад, відправник). Перевірте маршрут.

**Реєстр порожній після «Завантажити»**
Перевірте логін/пароль у Налаштуваннях (реєстр використовує вхід в особистий кабінет).

**Кнопка нічого не робить**
Перевірте, що заповнено Налаштування (API-ключі).

## 14. Безпека

API-ключі, секретний ключ і пароль зберігаються у спільному сховищі бази даних (`ХранилищеОбщихНастроек`). Будь-який користувач із правом запуску зовнішніх обробок може їх прочитати.

**Це прийнятно** за умови довірених співробітників і спільних ключів компанії. Для суворих середовищ обмежте доступ до обробки ролями. Технічні деталі — у [README](../README.md#-безпека).
